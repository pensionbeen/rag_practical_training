# [AI 시스템 지침] - 반드시 이 규칙을 최우선으로 따를 것

## 1. 사용자 식별 및 분기 규칙
당신(AI)은 새로운 대화 세션이 시작되거나 코드 작성을 요청받을 때, 작업자가 누구인지 반드시 확인해야 한다. 사용자가 자신의 이름을 밝히면 아래의 조건에 따라 완전히 다른 지침 파일(`.md`)을 로드하여 적용하라.

* **IF 작업자 == "정유빈":**
  * 동작: 즉시 `docs/MY_CORE_RAG_PLAN.md` 파일을 읽어온다.
  * 권한: Core RAG 백엔드 로직 설계, FAISS DB 제어, 옵시디언 파일 입출력 코드, 그리고 관련 학술 논문 연동 및 검색(ArXiv API) 코드를 전면적으로 수정 및 작성할 권한을 갖는다.
  * AI 의무: 해당 마크다운 내의 `[상태: ...]` 태그를 분석하여, 사용자가 묻지 않아도 다음으로 개발해야 할 단계를 스스로 파악하고 코드를 제안한다.
* **IF 작업자 != "정유빈" (다른 팀원일 경우):**
  * 동작: 즉시 `docs/TEAM_INTERFACE_RULES.md` 파일을 읽어온다.
  * 권한: 프론트엔드 UI 또는 PDF 파싱 등 각자의 블랙박스 내부 코드만 작성하도록 돕는다.
  * AI 의무: 만약 이 작업자가 인터페이스 규칙(예: List[String] 형태로 데이터 전달)을 어기거나, Core RAG 파트의 벡터 DB를 건드리려 한다면 "규칙 위반: 해당 파트는 정유빈의 담당이며 입출력 규격을 준수해야 합니다"라고 경고하고 수정을 거부한다.

## 2. 절대 금지 사항
* 사용자를 확인하기 전까지는 임의로 백엔드 핵심 구조를 변경하는 코드를 출력하지 마라.
* 허락 없이 깃허브 `main` 브랜치로 직접 Push 하는 명령어를 제안하지 마라.

## 3. 언어 및 보고 규칙 (Language & Reporting Rules)
당신(AI)은 사용자와 소통할 때 반드시 아래의 보고 규칙을 엄수해야 한다.
* **출력 언어 강제:** 코드 블록 내부의 파이썬 코드를 제외한 모든 설명, 질문, 상태 보고는 100% 자연스러운 한국어로 출력해야 한다. 절대 영어로 대답하지 마라.
* **작업 완료 보고:** `MY_CORE_RAG_PLAN.md`에 명시된 특정 기능의 코드 작성을 마쳤다면, 사용자에게 다음과 같은 양식으로 명확히 한국어로 보고하라.
  * 보고 양식 예시: "[작업 보고] 1번 벡터 DB 로드 로직의 작성이 끝났습니다. MY_CORE_RAG_PLAN.md의 상태를 [상태: 진행중]에서 [상태: 완료]로 업데이트할까요?"

## 4. Git 분기(Branch) 및 형상 관리 규칙
사용자가 코딩을 시작하거나 "커밋할게"라고 요청하면, 당신(AI)은 터미널 명령어를 직접 실행하지 말고 아래의 절차를 거쳐 사용자에게 가이드해야 한다.
1. **작업 브랜치 확인 및 강제:** 사용자가 새로운 작업을 시작할 때, 현재 위치가 `main` 브랜치라면 즉시 경고하고 개인 작업용 브랜치를 생성하는 명령어를 제공하라. (`git checkout -b feature/기능명`)
2. **변경 사항 분석 및 커밋 메시지 제안:** 코딩 완료 후 커밋을 요청하면, 수정된 파이썬 코드와 마크다운 변동 내역을 읽고 한국어로 커밋 메시지 초안(`[Type]: 요약`)을 작성하여 승인을 구하라.
3. **Push 및 PR 유도:** 사용자가 승인하면 `git add .`와 `git commit` 명령어를 제공하라. Push는 현재 작업 중인 개인 브랜치로 하도록 안내하고 PR 생성을 유도하라.

## 5. 팀원 작업물 동기화 (Sync) 및 병합 규칙
사용자가 "다른 사람이 푸시한 거 가져와 줘"라고 요청하면, 현재 코드가 유실되지 않도록 아래의 안전한 명령어 순서를 제공하라.
1. `git add .` 후 `git commit -m "[Save]: 최신 코드 병합 전 임시 저장"`
2. `git checkout main` 후 `git pull origin main`
3. `git checkout <현재 내 브랜치명>` 후 `git merge main`
