# 📋 옵시디언 연동형 초경량 RAG 시스템 작업 규칙서

## 1. 시스템 개요 (System Overview)
본 시스템은 외부 문서를 분석하여 사용자의 로컬 옵시디언(Obsidian) 지식 베이스에 마크다운 파일 형태로 자동 구조화하고, 해당 지식을 기반으로 출처(위치) 명시형 질의응답을 수행하며, 사용자가 이해하지 못한 개념을 다시 지식 베이스에 피드백하는 선순환 지식 관리 RAG 에이전트입니다.

---

## 2. 기술 스택 및 구동 환경 (Tech Stack)

* 프론트엔드 & 백엔드 통합: Streamlit (파이썬 단일 프레임워크)
  * 프론트엔드 표출 위치: 로컬 웹 브라우저 (http://localhost:8501)
  * 백엔드 구동 및 로그 위치: VS Code 가상환경(.venv) 터미널 콘솔창
* LLM 및 임베딩 엔진: Google Gemini API (gemini-1.5-flash)
  * 이유: 로컬 컴퓨터의 리소스 소모를 최소화하기 위해 클라우드 API 방식을 채택합니다.
* 벡터 데이터베이스: FAISS (로컬 인덱스 파일 저장 방식)
* 데이터 저장소: 사용자 컴퓨터 내의 실제 옵시디언 볼트(Vault) 로컬 폴더 경로

---

## 3. 핵심 기능별 세부 규칙 (Functional Rules)

### 📌 규칙 1: 문서 업로드 및 개념 추출 규칙 (인덱싱 단계)
* 입력(Input): 웹 UI를 통한 파일(PDF, TXT 등) 업로드
* AI 프롬프트 규칙: Gemini API 호출 시, 아래의 마크다운 출력 조건을 프롬프트에 강제합니다.
  - 당신은 지식 구조화 전문가입니다. 아래 입력된 문서에서 핵심 개념들을 추출하여 옵시디언 마크다운 양식으로 작성하세요.
  - 1. 중요도가 높은 개념은 개념명 옆에 [#중요] 태그를 붙이고 굵게 표시할 것.
  - 2. 일반 개념은 [#참고] 태그를 붙일 것.
  - 3. 다른 개념과 연관성이 식별되면 [[연관개념명]] 형태로 내부 링크를 걸 것.
* 출력(Output): 백엔드 파이썬 코드가 생성된 텍스트를 파싱하여 지정된 옵시디언 폴더에 [개념명].md 파일로 자동 저장 (open(path, 'w') 활용).

### 📌 규칙 2: 개념 검색 및 출처 추적 규칙 (조회 및 Q&A 단계)
* 작동 메커니즘: 기존 노트북 실습의 [분할 ➡️ 임베딩 ➡️ FAISS 저장 ➡️ 검색기 생성] 파이프라인을 그대로 계승합니다.
* 문서 로드 규칙: 옵시디언 폴더 내의 모든 .md 파일을 실시간으로 긁어모아 청크(Chunk)로 쪼갭니다.
* 출처(위치) 추적 규칙: FAISS 검색기(retriever.invoke(질문))가 유사 조각을 반환할 때, LangChain 문서 객체의 metadata['source'] 값을 추출합니다.
* UI 표출 규칙: 웹 브라우저 화면에 LLM의 최종 답변을 보여주기 전, "💡 본 답변의 근거가 된 옵시디언 노트: [파일명.md]"를 상단에 명시합니다.

### 📌 규칙 3: 미이해 개념 역저장 규칙 (피드백 루프 단계)
* UI 규칙: AI의 답변 하단에 [이해하기 어려움 - 복습 노트에 추가] 버튼을 배치합니다.
* 데이터 처리 규칙: 사용자가 해당 버튼을 클릭하면, 당시 질문과 AI설명을 묶어 옵시디언 폴더의 복습_필요_리스트.md 파일 맨 아래에 덧붙여 씁니다 (open(path, 'a') 활용).
* 세부 규칙: 파일 내부 추가 시 반드시 #복습필요 태그를 자동 기입합니다.

---

## 4. 개발 및 데이터 보안 규칙 (System Constraints)

* 보안 규칙 (API Key 노출 차단): 코드 내부에 Gemini API Key를 직접 하드코딩하는 것을 절대 금지합니다. 프로젝트 루트에 .env 파일을 만들고 키를 보관하며, .gitignore에 등록하여 GitHub Public 저장소에 유출되는 대참사를 원천 차단합니다.
* 가상환경 규칙: 모든 패키지 설치 및 앱 실행은 무조건 활성화된 (.venv) 가상환경 터미널 내부에서만 수행합니다.
* 경로 관리 규칙: 파이썬 코드 상의 모든 파일 경로 제어는 시스템 간 충돌을 막기 위해 파이썬 표준 라이브러리인 pathlib.Path 객체를 사용합니다.
